# Presentation Logic — 学术演讲深度内容构建

## 核心理念

PPT的本质不是slides，是**论证**。输出不是slide大纲，而是一份**完整的叙事文档**。用户读完后应该清楚知道整个演讲"在讲什么故事"。

## 前置条件

需要有已完成的文献精读笔记（来自 literature-review skill 或用户提供的材料）。

## Phase 1: 沉浸式重读

重新读完所有论文笔记。不是扫一眼，是逐篇读，边读边问自己：

**对每篇论文问3个问题：**
1. "这篇最让我意外的发现是什么？"
2. "这篇和其他论文矛盾吗？互补吗？"
3. "如果只能从这篇拿一个数据放进演讲，是哪个？"

读完后写一段**自由联想**（200-300字），不考虑结构，只写：
- 这些论文放在一起说明了什么？
- 领域里最大的认知转变是什么？
- 有什么是大家以为知道但其实是错的？

## Phase 2: 提炼核心论点

### 2.1 一句话主旨

格式：`[以前以为...] → [其实应该...] → [因为新证据表明...]`

### 2.2 支撑论点拆解

从主旨出发，拆成3-5个支撑论点。每个论点必须：
- 有唯一的核心主张
- 有至少一篇主力文献的具体数据支撑
- 有一个"听众可能的质疑"和你的回应

### 2.3 叙事弧线选择

**A. 认知颠覆型**（有争议或新突破）
```
"你以为的" → "但其实" → "证据是" → "所以应该"
```

**B. 层层递进型**（系统性综述型）
```
"问题" → "现有方法" → "不足" → "新方案" → "整合" → "实践"
```

**C. 侦探推理型**（单一核心研究的深度解读）
```
"线索" → "看似矛盾" → "真相揭示" → "回头看一切都说得通"
```

## Phase 3: 写叙事文档

**最重要的步骤。** 写连贯的、可以大声朗读的论证文字，不是bullet points。

### 文档结构

```markdown
# [演讲标题]

## 一句话主旨
> [Phase 2.1的结果]

## 第一部分：[标题]（预计X分钟，Y张slides）

### 论点
[2-3句话说清楚这一部分要传达什么]

### 论证
[连贯的文字，引用具体数据，有逻辑衔接]
[每段标注来源：(Author Year, 具体数字)]

### 🔍 自我质疑
**Q1: [听众可能问的尖锐问题]**
A: [回答，引用证据]

**Q2: [自己不太确定的地方]**
A: [诚实承认或引证回应]

### 过渡
[如何自然过渡到下一部分]

---

## 第N部分：[同样结构]

---

## 总结
### 核心信息（≤5条，每条一句话带数据）
### 未解决的问题（诚实列出）
### 临床/实践行动建议
```

### 写作规则

1. **禁止空话** — 每句话必须有信息量，不能只说"很重要"
2. **禁止同义反复** — 数据只在一个地方使用
3. **转折必须有新信息** — "但是"后面不能只是语气转折
4. **每段必须有文献支撑** — 无来源的主张标记[需验证]
5. **自我质疑必须尖锐** — 要能真正挑战论点的问题

## Phase 4: 多角度审查

写完后从三个视角重读：

### 视角1：严格的同行评审者
- 证据够吗？样本量够吗？是不是只有一项小研究？
- 有没有cherry-picking？反面证据被忽略了吗？
- 结果能推广吗？外部效度怎么样？

### 视角2：目标听众中的资深专家
- 跟临床经验一致吗？
- 我的科室能做到吗？需要什么条件？
- 具体cutoff/阈值/操作步骤是什么？明天能用吗？
- 跟现有指南矛盾吗？

### 视角3：准备提问的年轻医生/研究生
- 逻辑跟得上吗？哪里跳跃了？
- 术语太多了吗？
- 有没有假设了听众知道但其实不知道的背景知识？

**每个视角至少2-3个问题，在文档中回答。** 答不了的标记"需确认"并修改。

## Phase 5: 修改

基于审查修改文档：
- 删除重复
- 补充薄弱环节
- 调整逻辑顺序
- 确保自然过渡

## Phase 6: 输出

### 输出文件
- **`narrative_document.md`** — 存放在项目目录下
- 打开给用户审阅

### 完成标准
- **用户确认"可以"后，才能进入 html-slides skill**

## 质量检查清单

- [ ] 有明确的一句话主旨
- [ ] 每个论点有唯一主张，不重复
- [ ] 所有数据有来源（Author Year, PMID）
- [ ] 每部分有≥2个自我质疑
- [ ] 3个视角审查完成
- [ ] 部分之间有自然过渡
- [ ] 没有数据在多处重复
- [ ] "未解决的问题"是诚实的
- [ ] 用户已确认
